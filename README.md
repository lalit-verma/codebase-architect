# Codebase Analysis Framework

Evidence-first architecture analysis for any repository. Produces
documentation and a runtime harness optimized for **coding agent
efficiency** — enabling Claude Code, Codex, and Cursor to gain deep
codebase context at session start and write higher-quality code than
a cold start.

## What It Produces

The framework generates `agent-docs/` in your target repo:

```
agent-docs/
  agent-context-nano.md   ← inlined into system prompt via CLAUDE.md
  agent-context.md        ← curated context for Explore subagents
  patterns.md             ← full catalog of code patterns (>= 3 occurrences)
  routing-map.md          ← machine-readable task-to-doc routing (YAML)
  route-index.json        ← compiled routing substrate (generated by pensieve wire)
  agent-brief.md          ← compact architecture map
  agent-protocol.md       ← wiring instructions for agents
  index.md                ← navigation hub
  system-overview.md      ← top-level architecture
  decisions.md            ← architectural trade-offs
  glossary.md             ← project-specific terms
  uncertainties.md        ← unresolved questions
  subsystems/             ← per-subsystem deep dives
  structure.json          ← deterministic AST extraction (generated by pensieve scan)
  graph.json              ← cross-file dependency graph (generated by pensieve scan)
  structural-profiles.md  ← 7-layer structural summary (generated by pensieve scan)
```

**Load-bearing outputs:**

- **`agent-context-nano.md`** — ~40 line nano-digest inlined into the
  agent's system prompt (`CLAUDE.md` / `AGENTS.md` / `.cursorrules`).
  Handles common tasks without any tool-call reads. Contains the top
  2-6 pattern recipes verbatim.
- **`agent-context.md`** — ~120 line curated context file for Explore
  subagents. Handles medium/hard tasks where the nano-digest is
  insufficient.
- **`patterns.md`** — the complete durable catalog of all distinct
  confirmed patterns with 3 or more occurrences. Curation happens in
  `agent-context.md` / nano — `patterns.md` preserves the full set.
- **`routing-map.md`** + **`route-index.json`** — the routing substrate.
  Maps file paths, pattern names, and task keywords to subsystem docs.
  Consumed by the PreToolUse hook for path-aware routing hints.

## The 3-Phase Workflow

| Phase | Command | Runs | Output |
|-------|---------|------|--------|
| 1. Discover | `/analyze-discover` | Once | `system-overview.md`, `.analysis-state.md` |
| 2. Deep Dive | `/analyze-deep-dive` | Once per subsystem | `subsystems/{name}.md` |
| 3. Synthesize | `/analyze-synthesize` | Once | `agent-context.md`, `patterns.md`, `routing-map.md`, and all remaining docs |

Phase 2 runs multiple times — once per subsystem. Large subsystems
are recursively decomposed (up to depth 3). Each phase tells you what
to run next.

## Pensieve Toolkit

`pensieve` is a pip-installable Python CLI. The core commands (`scan`,
`brief`, `wire`) are fully deterministic — no LLM calls, pure transforms
from source code. The `benchmark` command calls an LLM judge for scoring.

```bash
pip install -e pensieve/python/    # or: uv tool install -e pensieve/python/
```

Commands:

| Command | What it does |
|---------|-------------|
| `pensieve scan <repo>` | Tree-sitter AST extraction → `structure.json`, `graph.json`, `structural-profiles.md` |
| `pensieve brief <paths>` | On-demand structural brief for specific files/directories (not persisted) |
| `pensieve wire --repo <repo>` | Inlines nano into `CLAUDE.md`, installs PreToolUse hook, builds `route-index.json` |
| `pensieve hook install/uninstall` | Standalone hook management |
| `pensieve benchmark generate/run` | Auto-benchmark suite for measuring agent efficiency |

`pensieve scan` supports 6 languages: Python, JavaScript, TypeScript,
Go, Java, Rust.

## Using It on a Repo

### 1. Install slash commands and shared resources (once per machine)

```bash
# Slash commands — user-level so they work in any repo
mkdir -p ~/.claude/commands
cp claude-code/commands/*.md ~/.claude/commands/

# Shared references
mkdir -p ~/.claude/codebase-analysis
cp -r shared/* ~/.claude/codebase-analysis/
```

### 2. Run structural extraction

```bash
pensieve scan /path/to/target-repo
```

This generates `structure.json`, `graph.json`, and
`structural-profiles.md` in the target repo's `agent-docs/`. The
slash commands use these as structural evidence during analysis.

### 3. Run the 3-phase analysis

Open Claude Code in the target repo:

```bash
cd /path/to/target-repo
claude
```

Then run each phase in order:

```
/user:analyze-discover
```
Wait for completion. Creates `system-overview.md` and the subsystem map.

```
/user:analyze-deep-dive
```
Run once per subsystem. Deep-dives each subsystem, writing docs to
`agent-docs/subsystems/`. For large subsystems, it proposes recursive
decomposition — approve or adjust.

```
/user:analyze-synthesize
```
Generates `agent-context.md`, `agent-context-nano.md`, `patterns.md`,
`routing-map.md`, and all remaining synthesis artifacts.

### 4. Wire the repo

```bash
pensieve wire --repo /path/to/target-repo
```

This does three things:
1. Inlines `agent-context-nano.md` into `CLAUDE.md` (idempotent, wrapped
   in markers)
2. Installs the PreToolUse hook at `.claude/hooks/pensieve-pretooluse.sh`
   and registers it in `.claude/settings.json`
3. Builds `route-index.json` from `routing-map.md`

### 5. Verify

```bash
# Hook installed
ls /path/to/target-repo/.claude/hooks/pensieve-pretooluse.sh

# Route-index built
python3 -c "import json; d=json.load(open('/path/to/target-repo/agent-docs/route-index.json')); \
  print(f'v{d[\"version\"]}: {len(d[\"subsystem_routes\"])} subsystems, {len(d[\"pattern_routes\"])} patterns')"

# Nano inlined
grep 'pensieve:nano:start' /path/to/target-repo/CLAUDE.md
```

After wiring, every Glob/Grep tool call in Claude Code gets path-aware
routing hints pointing to subsystem docs, pattern docs, or structural
zoom suggestions depending on the query.

## Runtime Harness

Once wired, the harness operates transparently during coding sessions:

**PreToolUse hook** — fires before every Glob/Grep. Injects a one-line
routing hint into Claude's context via the documented
`hookSpecificOutput.additionalContext` mechanism. The hint points to the
relevant subsystem doc and optionally suggests `pensieve brief <paths>`
for structural zoom.

**Routing priorities:**
1. **directory_prefix** — query path matches a subsystem's `owns_paths`.
   Includes `pensieve brief` suggestion when `brief_paths` exists.
2. **pattern_route** — query contains a pattern name or template
   basename (>= 8 chars for fragments). Doc-only.
3. **common_task** — query keywords overlap a subsystem's task list.
   Best-match-wins scoring. Includes `pensieve brief` suggestion when
   overlap is strong (>= 2 keywords) and `brief_paths` exists.
4. **fallback** — generic context pointer.

**`pensieve brief`** — on-demand structural primitive. Not persisted.
Generates a structural brief for any mix of file and directory paths,
showing signatures, internal dependencies, test mapping, entry points.
Recommended by the hook when a subsystem match is strong.

**Telemetry** — the hook appends to `agent-docs/hook-telemetry.jsonl`
with fields: `route_match_type`, `artifact_kind`, `brief_suggested`,
`target_subsystem`, `target_doc`, `session_id`. Append-only JSONL,
useful for measuring routing effectiveness.

## Other Platforms

### Codex

Copy `codex/AGENTS.md` to the target repo root. Paste prompts from
`codex/prompts/1-discover.md` through `3-synthesize.md` as tasks.
After synthesis, paste the nano-digest into `AGENTS.md`.

### Cursor

Copy `cursor/SKILL.md` to your Cursor skills location. The skill
auto-triggers on phrases like "analyze this codebase." After synthesis,
paste the nano-digest into `.cursorrules`.

Note: PreToolUse hook wiring and `pensieve wire` are Claude Code
specific. Codex and Cursor get the nano-digest + doc-reading path only.

## Re-running

All phases support re-runs on repos with existing `agent-docs/`. The
tool reads existing state and augments rather than overwrites.
`agent-context.md`, `agent-context-nano.md`, and `patterns.md` are
regenerated entirely on re-run. After re-running synthesis:

```bash
pensieve wire --repo /path/to/target-repo
```

This re-inlines the updated nano-digest and rebuilds the route-index.

## Evaluation

Two evaluation rubrics in `eval/`:

- **`eval/eval-toolkit.md`** — evaluates the framework itself (prompts,
  templates, references). 8 dimensions.
- **`eval/eval-output.md`** — evaluates the `agent-docs/` produced for
  a real repo. 10 weighted dimensions with spot-checks. Scored out of 32.

## Design Principles

- **Agent-first.** Primary consumer is a coding agent, not a human.
- **Read-only.** Never modifies source code.
- **Chat-first.** Presents findings for confirmation before writing.
- **Evidence-based.** Cites file paths. Labels uncertainty.
- **Pattern-aware.** Detects recurring code structures (>= 3 occurrences)
  and documents them as actionable recipes.
- **Recursive.** Large subsystems are decomposed up to depth 3.
- **Durable.** Output structured for long-term reuse. Supports re-runs.
